`buildcharts` is a CLI tool for defining build intent in `build.yml` and turning that metadata into a Docker Buildx Bake plan. You keep build definitions in your repo, pull reusable chart templates from an OCI registry, and run the same generated plan locally or in CI.

It sits between handwritten pipeline YAML and raw Dockerfiles. You declare what should happen, BuildCharts resolves the chart-backed implementation, and Docker Buildx Bake executes the generated plan.

## How it works

BuildCharts is designed for teams that want to standardize builds across many repositories without locking themselves into one CI provider. The model is simple:

- **Intent** stays in repository metadata (`build.yml`)
- **Implementation** lives in versioned OCI charts (`Chart.yaml`)
- **Execution** runs through Docker Buildx Bake (`.buildcharts/docker-bake.hcl`)

That gives you a portable build layer that can be debugged locally and reused across repositories.

## What problem it solves

BuildCharts reduces common CI/CD problems:

- Duplicated or nested pipeline YAML across many repositories
- Platform-specific pipeline logic that is hard to move between providers
- Fragile template rollouts where repositories drift onto different versions
- Poor local debuggability, where troubleshooting requires repeated CI runs

Instead of copying CI steps repo by repo, teams can centralize the build implementation in charts and let repositories consume versioned updates through normal dependency changes.

## What you define

```text
src/
├── build.yml            # Build metadata
├── charts/
│   └── buildcharts/
│       └── Chart.yaml   # Chart dependency data
```

## What BuildCharts generates

- `.buildcharts/docker-bake.hcl` from your metadata and chart dependencies

## Basic workflow

<Steps>
  <Step title="Create or scaffold build.yml and Chart.yaml">
    Run `buildcharts init` in a .NET repository to scaffold both files automatically.
  </Step>
  <Step title="Update chart lock (optional)">
    Run `buildcharts update` to resolve chart digests and write `Chart.lock` for reproducible builds.
  </Step>
  <Step title="Generate the Bake plan">
    Run `buildcharts generate` to validate metadata, pull charts, run plugins, and write `.buildcharts/docker-bake.hcl`.
  </Step>
  <Step title="Execute the plan">
    Run `docker buildx bake --file .buildcharts/docker-bake.hcl` to build all targets.
  </Step>
</Steps>
